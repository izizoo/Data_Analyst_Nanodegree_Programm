Data Audit:¶

Issues:¶

I start working on the dataset and exploring the data with focusing on the streets and postal codes and I notced many problems:

  • Street and Road names.
  • Postal codes.

Streets and Roads:¶

  • 1) St. and St (For street).
  • 2) Rd. and Rd (For Road).
  • 3) Some roads or streets have only the name (without specifying the type like "road,street,way").

Example on these:

  • 1) "Takhassusi St." fixed to "Takhassusi Street"
  • 2) "Omar Ibn Al Khattab Rd" fixed to "Omar Ibn Al Khattab Road"
  • 3) Can't be fixed because no one know if this is a road, street or a way! .

Postal codes:¶

  • Riyadh city must have postal codes of 5 numbers only (ex:12345).
  • 1) Some of postal codes in the data have more than 5 numbers like "12345 1234", the first part of this format is the postal code and the secound part of it which is 4 numbers is called the additional number.
  • 2) Some of postal codes in the data have less than 5 numbers like "1234", which is basically wrong and can't be fixed.

Example on these:

  • 1) "12783-8458" fixed to "12783".
  • 2) "7069" > Can't be fixed.

def is_street_name(address_key):
    return address_key == 'addr:street'

def isEnglish(s):
    try:
        s.encode(encoding='utf-8').decode('ascii')
    except UnicodeDecodeError:
        return False
    else:
        return True
#We check if the street name in arabic or english(arabic is ignored)    
    
def audit_street_type(street_name):
    """Check the street name """
    match = street_types.search(street_name)
    if match:
        street_type = match.group()
        if street_type not in expected and isEnglish(street_name):
            return update_street_name(street_name, street_mapping)


    return street_name

def update_street_name(name, mapping):
    """Fix the name and return or return it as it is if it's ok"""
    for key in mapping.keys():
        if re.search(key, name):
            name = re.sub(key, mapping[key], name)
            fixed_street_names.append(name)

    return name

def is_postal_code(address_key):
    return address_key == 'addr:postcode'
#we check first if the tag is a postal code




def audit_postal_code(postal_code):
    
    if len(postal_code) == 5 and postal_code.isdigit() :
        return postal_code

    bad_postal_codes.append(postal_code)

    # fix the code by taking only first 5 chars if it's more than 5 chars
    # this will not solve all problems but it'll reduce it.

    if len(postal_code) > 5 : 
        postal_code = postal_code[:5]
        fixed_postal_codes.append(postal_code)

    return postal_code

Suggestions:¶

After analyzing this data, I found out that all its problems because of human entry which consist of the format used for both streets and postal codes and the differint languages used in this data.

We could fix the data issues by some of suggestions I came up with:

  • Using standard language.
  • Standardizing the postal code and street names format.
  • Collecting the data from credible source.
